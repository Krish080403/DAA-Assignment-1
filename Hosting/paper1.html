Abstract

We present a depth-first search algorithm for generating all maximal cliques of an undirected graph, in which pruning methods are employed as in the Bronâ€“Kerbosch algorithm. All the maximal cliques generated are output in a tree-like form. Subsequently, we prove that its worst-case time complexity is O(3n/3) for an n-vertex graph. This is optimal as a function of n, since there exist up to 3n/3 maximal cliques in an n-vertex graph. The algorithm is also demonstrated to run very fast in practice by computational experiments.

How the Algorithm Works

The algorithm uses a depth-first search approach to find all maximal cliques in a graph. It starts with an empty set Q representing the current clique and recursively expands it by exploring candidate vertices from a set called SUBG. If SUBG is empty, Q is a maximal clique. Otherwise, each candidate vertex is added to Q, and the search continues within the subgraph induced by that vertex. To avoid redundant computations, the algorithm uses two pruning techniques: skipping already processed vertices and choosing a pivot vertex that reduces the search space. This efficient recursive process ensures all maximal cliques are generated without duplication.
